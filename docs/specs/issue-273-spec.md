# Spec: Issue #273 — Madness T3 (≥18) Replaces One Option with Unhinged Text

**Module**: docs/modules/conversation.md

---

## Overview

When the player's Madness shadow stat reaches threshold tier 3 (effective value ≥18), one of their dialogue options per turn must be flagged as "unhinged." This is a rules §7 shadow threshold effect — it degrades the player's conversation ability by replacing a random option with unhinged text generated by the LLM. The option retains its original stat for mechanical purposes (the roll still uses that stat), but the LLM is instructed to generate unhinged content for it. This completes the shadow T3 option-filter trifecta alongside the existing Fixation T3 (force last stat) and Denial T3 (remove Honesty) implementations.

## Input/Output Examples

### Example 1: Madness at 18 — one option marked unhinged

**Setup:**
- Player's `SessionShadowTracker` has Madness effective value = 18
- `ShadowThresholdEvaluator.GetThresholdLevel(18)` returns 3
- LLM returns 3 dialogue options: `[Charm "Hey there", Wit "Nice hat", Rizz "Come here often?"]`
- Dice roll for random index: `_dice.Roll(3)` returns 2 → index 1

**Result (TurnStart.Options):**
```
[0] DialogueOption { Stat=Charm, IntendedText="Hey there", IsUnhinged=false }
[1] DialogueOption { Stat=Wit, IntendedText="Nice hat", IsUnhinged=true }    ← replaced
[2] DialogueOption { Stat=Rizz, IntendedText="Come here often?", IsUnhinged=false }
```

The option at index 1 retains `Stat=Wit` (roll uses Wit) but is flagged `IsUnhinged=true`.

### Example 2: Madness at 17 — no options marked

**Setup:**
- Player's Madness effective value = 17
- `ShadowThresholdEvaluator.GetThresholdLevel(17)` returns 2 (below T3)

**Result (TurnStart.Options):**
```
[0] DialogueOption { Stat=Charm, IntendedText="Hey there", IsUnhinged=false }
[1] DialogueOption { Stat=Wit, IntendedText="Nice hat", IsUnhinged=false }
[2] DialogueOption { Stat=Rizz, IntendedText="Come here often?", IsUnhinged=false }
```

All options have `IsUnhinged=false`.

### Example 3: Only one option available — it becomes unhinged

**Setup:**
- Madness = 20 (T3)
- Only 1 option after Denial T3 filtering: `[Chaos "Yolo"]`
- `_dice.Roll(1)` returns 1 → index 0

**Result:**
```
[0] DialogueOption { Stat=Chaos, IntendedText="Yolo", IsUnhinged=true }
```

### Example 4: Madness T3 + Fixation T3 interaction

**Setup:**
- Madness = 18 (T3), Fixation = 18 (T3)
- Last stat used was Rizz
- LLM returns 3 options

**Processing order in StartTurnAsync:**
1. Fixation T3 runs first → all 3 options' stats become Rizz
2. Denial T3 runs → no effect (no Honesty to remove since all are Rizz)
3. **Madness T3 runs** → one random option marked `IsUnhinged=true`

**Result:** All options have `Stat=Rizz`, exactly one has `IsUnhinged=true`.

## Acceptance Criteria

### AC1: At Madness ≥18, one dialogue option is flagged `IsUnhingedReplacement = true`

The issue uses `IsUnhingedReplacement` but the contract (and consistent naming with other bool properties on `DialogueOption`) uses `IsUnhinged`. The property name should be `IsUnhinged`.

**Specification:**
- In `GameSession.StartTurnAsync()`, after the Denial T3 block and before the Horniness T3 block:
  1. Check if `shadowThresholds` contains `ShadowStatType.Madness` with value ≥ 3
  2. If yes, select a random index: `int idx = _dice.Roll(options.Length) - 1`
  3. Create a new `DialogueOption` copying all properties from `options[idx]` but with `isUnhinged: true`
  4. Replace `options[idx]` with the new instance
- Exactly one option is marked per turn when Madness T3 is active
- The random selection uses the injected `IDiceRoller` (not `System.Random`) for deterministic testing

### AC2: `DialogueContext` or the shadow taint block signals this to the LLM

- The `IsUnhinged` property on `DialogueOption` is readable by `ILlmAdapter` implementations via `DialogueContext.Options`
- `DialogueContext` already carries the options array — no additional signaling DTO changes are needed
- The existing shadow taint block in `SessionDocumentBuilder` (LlmAdapters) already includes Madness T3 flavor text when shadow thresholds are passed. The `IsUnhinged` flag on the option provides per-option specificity
- **No changes to `Pinder.LlmAdapters` are required in this issue** — the LLM adapter can read `IsUnhinged` from the options it receives. Prompt template updates for unhinged option text are a separate concern

### AC3: Tests — Madness=18 → one option marked; Madness=12 → no options marked

**Test cases required:**

1. **Madness=18 (T3):** Create a `GameSession` with player `SessionShadowTracker` where Madness effective value is ≥18. Call `StartTurnAsync()`. Assert exactly one option in `TurnStart.Options` has `IsUnhinged == true`. Assert all others have `IsUnhinged == false`.

2. **Madness=12 (T2):** Same setup but Madness=12. Assert no options have `IsUnhinged == true`.

3. **Madness=0 (no threshold):** Assert no options have `IsUnhinged == true`.

4. **Deterministic index selection:** Use a mock `IDiceRoller` that returns a known value for the `Roll(options.Length)` call. Verify the correct option index is marked.

### AC4: Build clean

- All existing tests (1146+) must continue to pass
- The new `isUnhinged` parameter default (`false`) ensures all existing `DialogueOption` constructors compile without changes
- No new NuGet dependencies

## Edge Cases

### Single option available
When `options.Length == 1` (e.g., after Denial T3 filtering removes all but one option), `_dice.Roll(1)` returns 1, index = 0. The sole remaining option becomes unhinged. This is correct — the player has no choice but to use the unhinged option.

### All three T3 thresholds active simultaneously
Processing order matters. The existing code runs Fixation T3 first, then Denial T3. Madness T3 runs last (after Denial T3). This means:
1. Fixation T3 forces all options to last-used stat
2. Denial T3 removes Honesty options (may be no-op if Fixation already changed stats)
3. Madness T3 marks one surviving option as unhinged

### Horniness T3 interaction
Horniness T3 (`_sessionHorniness >= 18`) forces all options to Rizz and runs **after** shadow threshold blocks. Madness T3 marks one option as unhinged **before** Horniness T3 runs. If Horniness T3 then replaces option stats, the `IsUnhinged` flag should be preserved. The Horniness T3 block creates new `DialogueOption` instances — it must copy the `isUnhinged` value from the original option.

**⚠️ Important:** The existing Horniness T3 block and Fixation T3 block both create new `DialogueOption` instances without passing `isUnhinged`. After this change, if Madness T3 runs before Horniness T3, the Horniness T3 reconstruction would lose the `IsUnhinged` flag. **However**, since Madness T3 runs in the shadow threshold block (before Horniness T3), and Horniness T3 creates fresh options, the implementer must ensure Horniness T3 preserves `IsUnhinged`. Two approaches:
1. Run Madness T3 **after** Horniness T3 (reorder the blocks)
2. Have Horniness T3 copy `IsUnhinged` when reconstructing options

**Recommended:** Run Madness T3 after all other option filters (Fixation T3, Denial T3, Horniness T3). Place it as the last option manipulation before returning `TurnStart`. This avoids the flag being lost by downstream reconstructions.

### Madness=18 with no `_playerShadows`
If `_playerShadows` is null (no shadow tracker configured), `shadowThresholds` is null, and the Madness T3 block is skipped entirely. No options are marked. This is correct.

### DialogueOption immutability
`DialogueOption` is immutable (all properties are `get`-only). The Madness T3 block must create a **new** `DialogueOption` instance with `isUnhinged: true`, copying all other property values from the original.

## Error Conditions

### No errors expected from this feature
- `_dice.Roll(n)` where `n >= 1` always returns a value in `[1, n]` — no out-of-range risk
- `shadowThresholds.TryGetValue` handles missing keys gracefully
- The `options` array is guaranteed non-empty at this point in `StartTurnAsync` (validated earlier)

### Potential issue: stale option references
If the host holds a reference to `TurnStart.Options` and later inspects it, the `IsUnhinged` flags are stable (immutable). No mutation risk.
