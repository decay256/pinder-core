## Constructing Test Fixtures

Many tests require a `RollResult` object. Since `RollResult` has a public constructor, tests can construct them directly:

```csharp
// RollResult constructor signature:
new RollResult(
    dieRoll: int,           // raw d20 (1-20)
    secondDieRoll: int?,    // null if no advantage/disadvantage
    usedDieRoll: int,       // the die used after adv/disadv
    stat: StatType,         // any StatType for testing
    statModifier: int,      // stat mod
    levelBonus: int,        // level bonus
    dc: int,                // target DC
    tier: FailureTier,      // failure tier (None for success)
    activatedTrap: null,    // null for most tests
    externalBonus: int      // default 0
)
```

To test **failure tiers**, construct a `RollResult` where `FinalTotal < DC` with the appropriate miss margin:

- **Fumble** (miss by 1–2): `tier: FailureTier.Fumble`
- **Misfire** (miss by 3–5): `tier: FailureTier.Misfire`
- **TropeTrap** (miss by 6–9): `tier: FailureTier.TropeTrap`
- **Catastrophe** (miss by 10+): `tier: FailureTier.Catastrophe`
- **Legendary** (nat 1): `tier: FailureTier.Legendary`, `usedDieRoll: 1`

To test **success scale**, construct a `RollResult` where `FinalTotal >= DC`:

- Beat by 1–4: e.g. `usedDieRoll: 15, statModifier: 0, levelBonus: 0, dc: 13` → FinalTotal=15, margin=2
- Beat by 5–9: e.g. `usedDieRoll: 20, statModifier: 0, levelBonus: 0, dc: 13, tier: FailureTier.None` (but not nat 20 — use `usedDieRoll: 18` + mods)
- Beat by 10+: e.g. `usedDieRoll: 18, statModifier: 5, levelBonus: 2, dc: 13` → margin=12
- Nat 20: `usedDieRoll: 20` → always +4 regardless of margin

To test **risk tier**, the need value is `dc - statModifier - levelBonus`:

- Safe (≤5): e.g. `dc: 13, statModifier: 5, levelBonus: 3` → need=5
- Medium (6–10): e.g. `dc: 13, statModifier: 3, levelBonus: 0` → need=10
- Hard (11–15): e.g. `dc: 13, statModifier: -2, levelBonus: 0` → need=15
- Bold (≥16): e.g. `dc: 20, statModifier: -1, levelBonus: 0` → need=21

## Edge Cases

1. **Boundary values for interest states**: Test the exact boundary between each state (0, 1, 4, 5, 9, 10, 15, 16, 20, 21, 24, 25). Ensure no off-by-one errors.

2. **Boundary values for failure tiers**: Miss margins at exact boundaries (1, 2, 3, 5, 6, 9, 10). These are tested via `RollResult` construction, not direct tier lookup.

3. **Boundary values for risk tiers**: Need values at exact boundaries (5, 6, 10, 11, 15, 16).

4. **Nat 20 with low total**: Nat 20 is always +4 on SuccessScale regardless of margin. A nat 20 where `FinalTotal < DC` should still be a success (auto-success rule).

5. **Nat 1 with high total**: Nat 1 is always Legendary failure regardless of total. A nat 1 where `FinalTotal >= DC` should still be a failure (auto-fail rule).

6. **ExternalBonus = 0 default**: All RollResult constructions in tests should use `externalBonus: 0` (the default) to avoid accidentally changing success/failure determination.

7. **RiskTierBonus on failure**: Even if RiskTier is Hard or Bold, `RiskTierBonus.GetInterestBonus` must return 0 on a failed roll.

8. **InterestMeter clamping**: `Apply(+100)` on a meter at 20 should cap at 25. `Apply(-100)` on a meter at 5 should floor at 0.

## Error Conditions

1. **Compilation errors from wrong method paths**: If any generated stub references a non-existent API (e.g. `LevelSystem.GetLevelBonus`), the build will fail. All paths must be corrected per the mapping table.

2. **RiskTierBonus null argument**: `RiskTierBonus.GetInterestBonus(null)` throws `ArgumentNullException`. Tests must not pass null.

3. **RollResult constructor validation**: The constructor does not validate ranges (e.g. `dieRoll` outside 1–20). Tests should use valid values to avoid testing constructor edge cases unrelated to rules.

4. **Namespace conflicts**: The new file must be in `Pinder.Core.Tests.RulesSpec` namespace. If another file already defines `RulesSpecTests` (none exists currently), there would be a conflict.
